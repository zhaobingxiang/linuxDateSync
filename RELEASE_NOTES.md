# DateSync 版本更新说明

---

## v1.1.0 - 2026-06-26

### 问题修复

1. **编辑任务保存时不再立即同步**
   - 修复了编辑任务保存后会立即触发一次同步的问题
   - 现在编辑保存后仅更新配置，不会执行同步操作

2. **定时同步时间精确到整分钟**
   - 修复了日/周/月计划同步时间不精确的问题
   - 现在同步会在目标时间的整分钟（00 秒）精确触发
   - 例如设置 09:00 同步，会在 09:00:00 触发，而不是保存时的秒数

3. **支持多个定时同步时间点**
   - 日计划、周计划、月计划现在支持添加多个同步时间点
   - 例如可以设置每天 09:00、12:00、18:00 分别同步
   - 界面提供添加/删除按钮管理多个时间点

4. **Root 密码字段优化**
   - 当用户名为 root 时，Root 密码输入框自动置灰禁用
   - 显示提示标签说明无需输入 Root 密码
   - 避免用户混淆，因为 root 用户登录不需要 sudo 权限

5. **同步进度回调错误修复**
   - 修复了同步时报错 `progress_callback() takes 2 positional arguments but 3 were given` 的问题
   - 现在同步过程正常运行，进度显示正常

6. **编辑已启用的定时任务**
   - 修复了编辑已启用定时同步的任务时，需要先取消勾选才能修改时间的问题
   - 现在可以直接编辑已启用任务的所有设置

### 功能优化

1. **配置兼容性**
   - 自动兼容旧版本配置文件格式
   - 旧配置中的单时间格式会自动转换为列表格式

2. **多任务同步优化**
   - 手动同步多个任务时，进度对话框不再阻塞界面
   - 可以同时进行多个任务的同步操作

3. **代码质量改进**
   - 移除已不再使用的冗余代码
   - 优化 SSH sudo 命令执行的提示符检测
   - 改进程序退出机制，使用标准的 Qt 退出方式

### 技术细节

- 数据模型：定时同步时间从单值改为列表存储
- UI 组件：使用 QListWidget 替代 QSpinBox 组合，支持多时间管理
- 信号处理：使用 blockSignals 模式避免信号级联触发
- 配置迁移：平滑兼容旧版本配置文件，无需手动修改

## v1.0.2 - 2026-06-23

### 核心功能修复

1. **同步计划时间触发优化**
   - 修复定时同步触发逻辑，从双向容差匹配改为精确时间匹配
   - 添加 sync_key 去重机制，防止同一时间点重复触发

2. **支持多时间点同步计划**
   - 月/周/日同步计划支持添加多个时间点
   - 时间选择界面从单组时/分输入改为列表式管理

3. **配置兼容性修复**
   - 新增配置字段格式兼容处理，自动将旧版单值时间格式转换为列表格式
   - 修复配置保存时字段名不一致导致的程序崩溃问题

### 安全与稳定性提升

4. **SSH 同步安全性增强**
   - 使用 `invoke_shell` 进行交互式 sudo 认证，替代明文密码传递
   - 改进临时文件生成方式，使用 `mktemp` 命令生成随机安全路径
   - 改进文件列表解析，使用 `find -print0` 方式处理特殊字符文件名

5. **密码安全存储**
   - 密码和 Root 密码采用基于机器标识的加密存储
   - 使用 `cryptography` 库的 Fernet 算法进行加密

6. **线程安全优化**
   - 使用 Qt 信号/槽机制实现跨线程 UI 更新，避免子线程直接操作 Widget

### 用户体验优化

7. **对话框布局优化**
   - 将固定尺寸对话框改为自适应布局 + 滚动区域
   - 解决切换同步类型后内容显示不全的问题

8. **月计划日期选择扩展**
   - 日期选择范围从 1-28 日扩展到 1-31 日
   - 添加大小月提示，选择 29-31 日时会提示在小月/2月可能自动跳过

9. **校时功能增强**
   - 添加 NTP 校时超时控制和多服务器重试机制
   - 增加 SetLocalTime 返回值检查，失败时记录详细错误信息

10. **编辑任务后不立即同步**
    - 更新配置时保留原 `last_sync` 时间，避免保存后立即触发同步

### 代码质量提升

11. **依赖清理**
    - 移除未使用的 `schedule` 包依赖
    - 清理重复导入和无效代码

12. **日志级别控制**
    - 支持运行时动态调整日志级别（DEBUG/INFO/ERROR）
