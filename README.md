# DateSync - Linux 目录同步工具

一款用于将 Linux 服务器目录同步下载到本地 Windows 机器的图形化工具。

## 功能特性

- **SSH 文件同步**：通过 SSH 协议连接 Linux 服务器，递归同步下载文件和目录
- **Root 权限支持**：可选填写 Root 密码，自动使用 sudo 读取无权限文件
- **多服务器管理**：支持同时配置和管理多个服务器同步任务
- **灵活的定时同步**：
  - 时间间隔：每隔 N 分钟自动同步
  - 日计划：每天在指定时间同步，支持设置多个时间点
  - 周计划：每周指定日期和时间同步，支持多个时间点
  - 月计划：每月指定日期和时间同步，支持 1-31 日及多个时间点
- **服务器在线检测**：自动检测服务器连接状态，实时显示在线/离线
- **系统校时**：从多个 NTP 服务器同步 Windows 系统时间
- **开机自启动**：支持设置程序开机自动运行
- **系统托盘**：最小化到系统托盘后台运行，双击恢复窗口
- **密码加密存储**：服务器密码使用基于机器特征的加密存储，配置文件中不暴露明文

## 快速开始

### 方式一：直接运行可执行文件（推荐）

```bash
dist_new/DateSync.exe
```

### 方式二：使用 Python 源代码运行

```bash
# 进入项目目录
cd d:\ide\datesync

# 创建并激活虚拟环境
python -m venv .venv
.venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 运行程序
python main.py
```

## 使用说明

### 添加同步任务

1. 点击「添加任务」按钮
2. 填写服务器信息：
   - 任务名称（用于标识）
   - 服务器地址（IP 或域名）
   - SSH 端口（默认为 22）
   - 用户名和密码
   - Root 密码（可选，用于读取无权限文件）
   - 远程目录（要同步的 Linux 目录路径）
   - 本地目录（保存到 Windows 的路径）
3. 设置定时同步（可选）：
   - 勾选「启用定时同步」
   - 选择同步类型：时间间隔 / 日计划 / 周计划 / 月计划
   - 设置具体时间（日/周/月计划支持添加多个时间点）
4. 点击「确定」完成添加

### 手动同步

1. 勾选要同步的任务
2. 点击「立即同步」按钮
3. 等待同步完成，查看同步结果

### 定时同步说明

- **时间间隔**：每隔指定分钟数自动同步一次
- **日计划**：每天在设定的时间点自动同步，可添加多个时间点（如 09:00、12:00、18:00）
- **周计划**：每周在选定的星期和时间自动同步，同样支持多个时间点
- **月计划**：每月在选定的日期和时间自动同步，支持 1-31 日（29-31 日在小月/2 月会自动跳过）

定时同步在目标时间的整分钟（00 秒）精确触发，编辑任务保存后不会立即触发同步，会等待下一个周期。

### Root 权限同步

如果远程目录中有普通用户无权读取的文件，填写 Root 密码后程序会自动使用 sudo 读取这些文件。密码通过安全方式传递，不会出现在进程列表中。

### 系统校时

1. 点击「校时」按钮
2. 程序依次尝试连接多个 NTP 服务器
3. 校时成功或失败会显示相应提示

注意：校时功能需要管理员权限运行。

## 安全说明

1. **密码存储**：服务器密码和 Root 密码使用基于机器特征的 Fernet 加密存储在配置文件中，非明文保存
2. **网络通信**：所有数据传输使用 SSH 协议加密
3. **Sudo 执行**：Root 密码通过交互式 Shell 的标准输入传递，不会暴露在命令行参数或进程列表中
4. **临时文件**：使用 sudo 下载文件时，远程临时文件通过 mktemp 生成随机路径，避免路径预测风险

## 常见问题

### Q: 同步失败怎么办？

请检查：
- 服务器地址和端口是否正确
- 用户名和密码是否正确
- 网络连接是否正常
- 远程目录是否存在且有读取权限（无权限文件需填写 Root 密码）
- 本地目录是否有写入权限

### Q: 定时同步不执行怎么办？

请检查：
- 定时同步功能是否已勾选启用
- 系统时间是否正确（可使用校时功能校准）
- 程序是否在运行中（可以在系统托盘查看）

### Q: 编辑任务后立即同步了一次？

此问题已在当前版本修复，编辑保存任务后不会立即触发同步，会等待下一个正常周期。

### Q: 定时同步的时间不精确？

此问题已在当前版本修复，日/周/月计划现在在目标时间的整分钟（00 秒）精确触发。

### Q: 程序最小化后找不到了？

程序最小化到系统托盘了，双击托盘图标即可恢复窗口。

### Q: 校时失败怎么办？

请检查：
- 网络连接是否正常
- 防火墙是否阻止了 NTP 连接（端口 123）
- 是否以管理员权限运行程序

### Q: 杀毒软件报病毒怎么办？

这是误报。本软件使用 PyInstaller 打包，可能被某些杀毒软件的启发式检测误判。你可以：
1. 将 `DateSync.exe` 添加到杀毒软件的信任列表
2. 向杀毒软件厂商提交误报反馈
3. 使用源代码方式运行
